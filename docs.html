API Endpoints

POST /analyze/resume ATS Analysis
Analyze a resume for ATS compatibility. Returns a 0–100 score, letter grade, list of issues, extracted skills, and improvement suggestions.

Body: { resume_text: string }
POST /analyze/resume { "resume_text": "Jane Smith... } // Response { "ats_score": 85, "grade": "A", "issues": [], "skills": {...} }
POST /analyze/job-match Job Match Score
Compare a resume against a job description. Returns keyword match score, matched/missing keywords, seniority level, and a hiring recommendation.

Body: { resume_text, job_description }
POST /analyze/job-match { "resume_text": "...", "job_description": "..." } // Response { "match_score": 72, "match_level": "Good Match", "missing_keywords": [...] }
POST /analyze/full ★ Premium
Combined analysis in one call: ATS score + job match + skills + prioritized suggestions. The main endpoint for most integrations.

Body: { resume_text, job_description? }
POST /analyze/full { "resume_text": "...", "job_description": "..." // optional } // Response { "overall_score": 79, "grade": "B", "ats_analysis": {...}, "job_match": {...}, "suggestions": [...] }
POST /extract/skills Skill Extractor
Lightweight endpoint that extracts technical and soft skills from any resume text. Fast and low-cost — ideal for high-volume pipelines.

Body: { resume_text: string }
POST /extract/skills { "resume_text": "..." } // Response { "technical_skills": [ "python", "react", ... ], "soft_skills": [...] }

Step-by-Step Guide

How to list on RapidAPI

1
Deploy your API to a live server
RapidAPI needs a live URL — your API can't run on localhost. The fastest free option is Railway:
npm install -g @railway/cli railway login railway init # creates a new project railway up # deploys your code railway domain # generates a public URL like https://myapp.railway.app
Alternatively, push to GitHub and deploy on Render.com (free tier, no CLI needed).
2
Create a RapidAPI provider account
Go to rapidapi.com → Sign up → After login, click your avatar → "Add New API". Choose "REST API" and give it a name like "Resume Analyzer & Job Match API".
3
Set the Base URL and import your spec
In the Hub Listing page → Definitions tab:
• Set Base URL to your Railway/Render URL (e.g. https://yourapp.railway.app)
• Click "Import OpenAPI file" and upload openapi.yaml — this auto-creates all your endpoints with descriptions.
4
Set up the proxy secret (security)
In your RapidAPI dashboard → Security tab, copy your X-RapidAPI-Proxy-Secret. Set it as an environment variable on Railway:
railway variables set RAPIDAPI_PROXY_SECRET=your-secret-here
This prevents people from bypassing RapidAPI and calling your server directly.
5
Configure pricing tiers
Go to the Monetization tab in your API listing. Create 4 plans matching the pricing table above: Free (10 req), Basic ($9.99/500 req), Pro ($29.99/5000 req), Ultra ($99.99/unlimited). Set /analyze/full as available on Basic and above if you want to gate your best endpoint.
6
Write your listing & publish
Fill in the About tab: write a clear description, list use cases (job boards, HR tools, resume builders), add your logo. Then toggle "API is Public" to go live. You'll appear in RapidAPI's marketplace within a few hours.

📣 Tip: Post about your API on dev.to, Reddit r/webdev, and LinkedIn to drive initial traffic and reviews — the first 5-star reviews dramatically boost discoverability.
